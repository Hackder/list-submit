1. RingBuffer

V tejto úlohe naprogramujete vlastnú polymorfnú štruktúru podobnú (nie nutne rovnakú) Ringbufferu: https://en.wikipedia.org/wiki/Circular_buffer . Ide o cyklickú pamäť, ktorá má síce zadanú veľkosť, avšak pri jej zaplnení je možné vložiť nové dáta a to vymazaním starých. Nepoužijeme presnú definíciu, ale jemne si ju upravíme.

Poznámka: Pri riešení vysoko(!) odporúčame udržiavať si nejaký index sledujúci vkladacie zmeny v poli (v popise funkcií nižšie je vždy napísané, čo sa s ním deje - daný index je vždy podčiarknutý ). Pri operáciach nezabúdajte, že pole je kruhové - ak by mal index vyletieť mimo poľa, tak sa namiesto toho nastaví na nulu (teda na začiatok).

V triede RingBuffer naprogramujte metódy:

  • RingBuffer(int kapacita) - vytvorí nové klasické(!!) pole o veľkosti kapacita, index nainicializujte na nulu
  • public int size() - vráti počet ne-null-ových prvkov v poli (so skutočnými hodnotami, čo sa môže často líšiť od kapacity)
  • public T top() - vráti posledne vložený prvok v poli, príp. null, ak bol tento prvok práve nahradený, či pole je prázdne; implementačne povedané, je to vždy prvok priamo pred indexom (index -1)
  • public T get(int index) - vráti prvok na danom vstupnom indexe (argumentu funkcie), príp. null
  • public T[] get(int index, int pocet) - vráti prvky (vrátane null hodnôt) v poradí podľa vstupného indexu (argumentu funkcie); od vstupného indexu presne pocet hodnôt
  • public boolean push(T hodnota) - pridá prvok do poľa na najbližšie voľné (nullové) miesto na základe indexu (nie od nuly, pretože v kruhom poli 'teoreticky' nie je začiatok 0); ak je pole už plné, nič nepridá; vráti, či sa hodnota úspešne pridala; pri každom posune sa posunie aj index
  • public int push(T[] hodnoty) - pridá prvky do poľa na najbližšie voľné (nullové) miesta na základe indexu ; ak je pole už (pred/počas procesu pridávania) plné, viac prvkov nepridá; vráti počet úspešne pridaných prvkov; pri každom posune sa posunie aj index
  • public T[] pushForce(T[] hodnoty) - pridá prvy do poľa na najbližšie miesta na základe indexu ; ak sa na danom mieste už nachádza hodnota, nahradí ju novou; pri každom posune sa posunie aj index ; vráti pole !iba! vymazaných (nahradených, teda pôvodne nenullových) prvkov v poradí podľa indexu
  • public T[] remove(int index, int pocet) - vymaže (nahradí null hodnotou) prvky začínajúce na mieste vstupného indexu (argumentu funkcie), ak narazí na prázdne miesto počas vyhadzovania prvkov, tak ďalej nepokračuje (teda môže odstrániť aj menej prvkov ako pocet), inak odstráni presne pocet prvkov; index sa pri tomto úkone vôbec nemení; vráti vymazané prvky v poradí podľa vstupného indexu (argumentu funkcie)

Príklad: (v komentári je stav po danej operácii):

RingBuffer <Integer >r = new RingBuffer <Integer >(3);      // {null, null, null}; index=0
r.push(20);                                              // {20, null, null}; index=1
r.push(30);                                              // {20, 30, null}; index=2
System.out.println(r.top());                             // 30
r.push(40);                                              // {20, 30, 40}; index=0
System.out.println(r.size());                            // 3         
r.remove(0, 2);                                          // {null, null, 40}; index=0
r.push(new Integer[]{5, 10, 15, 20, 25, 30, 35});        // {5, 10, 40}; index=2
r.remove(1,1);                                           // {5, null, 40}; index=2
System.out.println(r.top());                             // null
System.out.println(Arrays.toString(r.get(2, 2)));        // {40, 5}
r.pushForce(new Integer[]{100, 200});                    // {200, null, 100}; index=1

2. Fruit Binary Tree

Zadaný máte binárny strom triedy Tree , ktorý obsahuje koreň binárneho stromu triedy Node . Každý Node sa ako na strome rozvätvuje napravo a naľavo. Zároveň na ňom môže rásť ovocie rôzneho druhu.  

                                    class Tree{

                                    Node root;
}
                                
                                    public class Node {

                                    Node left;

                                    Node right;

                                    Object[] fruits;
}

Premenná fruits obsahuje všetky druhy ovocia, ktoré na danom Node rastú, napr. new Object[]{new Apple(), new Peach()...} . Triedy rôznych typov ovocia neimplementujte!

Vašou úlohou je doimplementovať triedy Tree a Node . V triede Tree zároveň implementujete funkcie:

public boolean isOneTypeOfFruitOnNode() - funkcia vráti true, ak každý Node v Tree má vo svojom liste objektov iba jeden typ ovocia
 napr. ak sa na Node nachádzajú iba jablká t.j. new Object[]{new Apple(), new Apple()}, tak vráti true

public boolean isSameOneTypeOfFruitOnChildren() - funkcia vráti true, ak deti každého Node v strome (čiže left Node a right Node) majú rovnaký jeden typ ovocia (čiže ich listy objektov majú rovnaký typ ovocia)
napr. ak sú na left Node len jablká (Object[]{new Apple(), new Apple()}) a na right Node tiež len jablká (Object[] {new Apple(), new Apple()}), vráti true
Hint: na zisťovanie typov sa používa getClass()

Priklady

Tree t = new Tree(new Node(new Node(null, null, new Object[]{new Coconut(), new Apple()}), new Node(null, null, new Object[]{new Pear(), new Coconut()}), new Object[]{new Banana(), new Orange()}))
t.isOneTypeOfFruitOnNode() ->false
t.isSameOneTypeOfFruitOnChildren() ->false

Tree t = new Tree(new Node(new Node(null, null, new Object[]{new Strawberry(), new Strawberry()}), new Node(null, null, new Object[]{new Apricot(), new Apricot()}), new Object[]{new Apricot(), new Apricot()}))
t.isOneTypeOfFruitOnNode() ->true
t.isSameOneTypeOfFruitOnChildren() ->false

Tree t = new Tree(new Node(new Node(null, null, new Object[]{new Grape(), new Apricot()}), new Node(null, null, new Object[]{new Peach(), new Pear()}), new Object[]{new Plum(), new Orange()}))
t.isOneTypeOfFruitOnNode() ->false
t.isSameOneTypeOfFruitOnChildren() ->false

Tree t = new Tree(new Node(new Node(null, null, new Object[]{new Strawberry(), new Strawberry()}), new Node(null, null, new Object[]{new Strawberry(), new Strawberry()}), new Object[]{new Pear(), new Pear()}))
t.isOneTypeOfFruitOnNode() ->true
t.isSameOneTypeOfFruitOnChildren() ->true

3. Rhododendron Tree

Rhododendrom je strom, nie síce v našich končinách, ale dole v Azii, ktorý sa vyznačujte zvlástnym vetvením konárov. Kým "normálny" strom trochu pripomína informatický binárny strom, kedy z vetvy sa odštiepi iná vetva, tak rhododendrom je strom, kde každý vrchol môže mať ľubovoľný počet podstromov. Teda 0, 1, 2 ale aj 3, 4, ... Na reprezentáciu takéhoto stromu potrebujeme štruktúru podobnú Nod, ktorá obsahuje hodnotu a zoznam synov daného vrchola. Zrejme ste sa stretli s binárnym stromom, ten mal synov dvoch, možno najviac dvoch, a tento ich má ľubovoľne veľa. V tomto strome ideme vyhľadávať, preliezať a takéto stromy môžeme navzájom aj porovnávať. V priloženom template súbore Nod.java máte príklad rekurzie, ktorá vzorovo prelieza rhododendron strom, a čaká, že null môže byť kdekoľvek. T je typový parameter generickej triedy a extends Comparable <T > znamená, že ním môže byť ľubovoľný typ, ktorý má definovaný interface Comparable. Vašou finálnou úlohou je porovnávať stromy, preto trieda implementuje interface Comparable <Nod <T >>, ktorý musíte doprogramovať vy. Pri porovnávaní celých stromov musíme vedieť porovnať aj hodnoty vo vrcholoch, preto na typ T je kladený predpoklad extends Comparable <T >, čo znamená, že hodnoty typu T vieme porovnávať.

Pre triedu Nod definujte:

- @Override public String toString () pomôže vám pri ladení ďalšieho kódu

a chýbajúce metódy pre interejsy Comparable <Nod <T >> , RTree <T > tak, aby
- contains (T elem) vrátilo true, ak prvok s hodnotou elem sa v strome nachádza, inak false,
- isLeaf (T elem) vráti true, ak list s hodnotou elem sa nachádza niekde v strome, inak false. List je vrchol, ktorý už nemá žiadne podstromy, teda vrchol, ktorého zoznam synov je null, resp. prázdny zoznam. 

- List <T >preorder () vráti zoznam všetkých prvkov stromu nájdených v preorderovom poradí,
 - @Override public int compareTo (Nod <T >o) porovná strom this so stromom o . Stromy sa porovnávajú cez porovnanie hodnôt vrcholov v preorderovom poradí a to takto:
+ ak sú oba null, výsledok je 0,

+ null je menší ako nenull, odtiaľ výsledok >0 / <0, podobne ako compareTo,

+ ak sú oba nenull, porovnávajú sa hodnoty value v nich, ak porovnanie rozhodne v prospech niektorého, tak to je výsledok celého porovnávania stromov,

+ ak porovnanie nerozhodne (teda this.value.compareTo(o.value())==0), porovnávajú sa rekurzívne všetky zodpovedajúci synovia v poradí 0,1,2,... jedného a druhého (pozor, nemusí ich byť rovnaký počet)

+ kratší zoznam synov je doplnený null-mi

+ zodpovedajúci synovia sa porovnávajú, kým porovnanie nerozhodne v prospech niektorej strany. Vtedy porovnávanie končí, a ďalší synovia sa už neporovnávajú.

+ ak aj všetci synovia sú rovnakí, oba stromy sú rovnaké, a teda výsledkom je 0.
